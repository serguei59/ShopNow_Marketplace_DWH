# 3. Objectifs de la mission

Les objectifs sont structurés en trois axes :

## 3.1. Axe technique
- Concevoir une architecture Cloud cible.
- Implémenter un socle Data Factory + Event Hubs + Storage.
- Mettre en place la supervision, le logging et le DRP.
- Standardiser la sécurité (IAM, RLS, cloisonnement vendeur).

## 3.2. Axe Data
- Définir la modélisation SCD2.
- Concevoir le DWH logique.
- Assurer la qualité (règles, checks automatiques).
- Gérer la multi-source et la multi-fréquence.

## 3.3. Axe gouvernance
- Définir les rôles/permissions.
- Structurer le modèle multi-vendeur.
- Cartographier les preuves demandées pour la certification (E6).
