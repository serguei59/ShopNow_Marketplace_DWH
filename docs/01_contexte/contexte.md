# 1. Contexte général

L’entreprise souhaite mettre en place une plateforme de type **Data Marketplace** destinée à exposer des jeux de données multi-sources (internes + partenaires).  
L’objectif est d’offrir un accès unifié, sécurisé et supervisé aux données tout en garantissant l’autonomie des équipes métiers.

Les enjeux clés du contexte actuel :

- Hétérogénéité des sources de données.
- Absence d’un socle commun de qualité, logs, supervision.
- Gouvernance limitée pour le multi-fournisseur.
- Pas de standardisation des mécanismes MCO.
- Besoin d’un modèle de réplication / historisation (SCD2).
- Nécessité d’industrialiser les pipelines existants.

La mission vise à concevoir un socle robuste, industrialisable et conforme aux meilleures pratiques Cloud & Data Engineering.
